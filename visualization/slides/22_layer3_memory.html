Design Rationale
  • 4-layer memory. From short-term logs to long-term patterns.
  • Enough context to summarize; timely enough to be useful.
  • Overwrite if only the latest value matters (current state, pattern); append if history accumulates (logs, highlights, states) — then consolidate periodically to bound growth.
  • Direct write to OpenClaw's memory directory.
